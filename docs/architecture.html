Data flow — a filtered view request

1

User interacts with FilterBar

The FilterBar component updates shared state in useFilters.js. Refs (selectedWarehouse, selectedCategory, …) change reactively across all views.

2

View triggers an API call

A view (e.g. Dashboard.vue) watches filter changes and calls api.getDashboardSummary(filters). The centralised client in client/src/api.js serialises filters into query params.

3

HTTP request crosses to FastAPI

axios sends GET /api/dashboard/summary?warehouse=…&category=… to port 8001. CORS is permissive for local development.

4

Filters apply against in-memory lists

apply_filters() and filter_by_month() in main.py walk the in-memory lists. Quarters (Q1-2025 … Q4-2025) expand to their three component months via QUARTER_MAP.

5

Pydantic validates the response

The endpoint returns aggregates (total inventory value, low-stock count, pending orders, …). For typed endpoints, the response_model ensures the shape matches the declared model.

6

Vue updates reactive state

The view writes the response into a ref (e.g. allOrders). Computed properties derive everything else — charts, totals, formatted strings — and the DOM re-renders.
